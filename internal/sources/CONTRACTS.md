# Source Contracts

This document describes the contracts and semantics for wallpaper source implementations in Walens.

## Overview

Sources are plugins that fetch image metadata from external wallpaper providers (e.g., Gelbooru, Wallhaven, etc.). The source system uses an iterator-based contract to support progressive downstream processing and early cancellation.

## Core Interfaces

### Source Interface

```go
type Source interface {
    TypeName() string
    DisplayName() string
    ValidateParams(raw json.RawMessage) error
    ParamSchema() *huma.Schema
    DefaultLookupCount() int
    Fetch(ctx context.Context, req FetchRequest) iter.Seq2[ImageMetadata, error]
    BuildUniqueID(item ImageMetadata) (string, error)
}
```

## FetchRequest Contract

`FetchRequest` contains execution parameters passed to `Source.Fetch`.

### Fields

| Field | Type | Description |
|-------|------|-------------|
| `Params` | `[]byte` | Source-specific configuration as raw JSON (user-configured params from the sources table) |
| `LookupCount` | `int` | Upstream lookup budget |

### Lookup Budget Semantics

**Critical: `LookupCount` is an upstream budget, not a guaranteed result count.**

- `LookupCount` specifies how many items the source should **attempt to check** from the upstream service
- The actual number of `ImageMetadata` items yielded may be **less than** `LookupCount`
- **Skipped items count toward the budget**: non-image posts, filtered content, etc. all consume lookup budget
- **Deduplicated items count toward the budget**: items already known to the system still count
- Sources should request items from upstream until the budget is exhausted or context is cancelled

Example: If `LookupCount = 100`:
- Source may fetch 100 posts from API
- 40 are videos (skipped, count toward budget)
- 20 are duplicates (count toward budget)
- 40 are valid images (yielded to caller)
- Result: 40 images from a budget of 100

## ImageMetadata Contract

`ImageMetadata` represents metadata for a single image item from a source.

### Required Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `UniqueIdentifier` | `string` | **Yes** | Source-specific string that uniquely identifies this image within the source. Used as input to `BuildUniqueID`. Must be stable across fetches for the same image. |
| `SourceItemID` | `string` | **Yes** | The external source's native unique identifier (e.g., post ID, image hash). Used as input to `BuildUniqueID`. |

### Optional Fields

| Field | Type | Description |
|-------|------|-------------|
| `PreviewURL` | `string` | URL to a smaller/preview version for gallery display |
| `OriginURL` | `string` | URL to the original/full-resolution image file for download |
| `OriginalID` | `string` | Original/uploader-assigned identifier (may be empty) |
| `Uploader` | `string` | Username of the person who uploaded this image |
| `Artist` | `string` | Artist/creator associated with this image |
| `MimeType` | `string` | MIME type (e.g., "image/jpeg", "image/png", "image/webp") |
| `FileSizeBytes` | `int64` | Size of original image file in bytes |
| `Width` | `int` | Pixel width of the image |
| `Height` | `int` | Pixel height of the image |
| `AspectRatio` | `float64` | Width/height ratio (e.g., 1.777778 for 16:9) |
| `IsAdult` | `bool` | Whether image contains adult content |
| `Tags` | `[]string` | Tags/categories associated with this image |

**Note**: At least one of `PreviewURL` or `OriginURL` should be populated.

## Unique Identifier Generation

### BuildUniqueIDInputs

The `BuildUniqueID` method uses the following required fields from `ImageMetadata`:

- **`SourceItemID`**: The external source's native identifier
- **`UniqueIdentifier`**: The source-specific unique string for this image

### Requirements for Generated IDs

The unique ID generated by `BuildUniqueID` must be:

1. **Stable**: Same `ImageMetadata` inputs always produce the same output
2. **Globally Unique**: Incorporate source identity to avoid collisions across different sources
3. **Deterministic**: No randomness in ID generation

### Recommended Pattern

```go
func (s *MySource) BuildUniqueID(item ImageMetadata) (string, error) {
    // Combine source type with required fields
    input := fmt.Sprintf("%s:%s:%s", s.TypeName(), item.SourceItemID, item.UniqueIdentifier)
    // Use consistent hash
    hash := sha256.Sum256([]byte(input))
    return hex.EncodeToString(hash[:]), nil
}
```

## Iterator Contract

The `Fetch` method returns `iter.Seq2[ImageMetadata, error]`, which is a Go iterator that yields pairs of items and errors.

### Requirements

1. **Lazy Evaluation**: Items should be yielded progressively as they are fetched from upstream, not buffered entirely before iteration begins

2. **Context Cancellation**: The iterator must promptly honor `context.Context` cancellation. When the context is cancelled, the iterator should stop fetching and return

3. **Early Stop Support**: Callers may stop iteration early without requiring the iterator to drain remaining items or return an error. The iterator should not block or error on early stop

4. **Error Surfacing**: Iteration errors should be yielded as they occur via the error channel, not accumulated and returned at the end

### Example Iterator Implementation

```go
func (s *MySource) Fetch(ctx context.Context, req FetchRequest) iter.Seq2[ImageMetadata, error] {
    return func(yield func(ImageMetadata, error) bool) {
        // Process in batches
        offset := 0
        for {
            // Check context cancellation
            if err := ctx.Err(); err != nil {
                return
            }

            // Fetch batch from upstream
            batch, err := s.fetchBatch(ctx, req.Params, offset, req.LookupCount)
            if err != nil {
                yield(ImageMetadata{}, err)
                return
            }

            // Yield each item
            for _, item := range batch.Items {
                // Transform to ImageMetadata
                meta := transformToImageMetadata(item)
                
                // Yield item; if caller stops (returns false), stop iterating
                if !yield(meta, nil) {
                    return
                }
            }

            // Check if we've exhausted budget
            offset += len(batch.Items)
            if offset >= req.LookupCount || !batch.HasMore {
                return
            }
        }
    }
}
```

### Usage Pattern

```go
// Caller usage of Fetch iterator
for meta, err := range source.Fetch(ctx, req) {
    if err != nil {
        // Handle error (may continue or abort)
        log.Printf("fetch error: %v", err)
        continue
    }
    
    // Process meta
    // May break early without error
    if someCondition {
        break
    }
}
```

## Source Registration

Sources implement the `Source` interface and register themselves via the global registry:

```go
// In source implementation init()
func init() {
    registry.Register(&MySource{})
}
```

Registration happens automatically via `init()` functions. The `TypeName()` must be unique across all registered sources.

## Lifecycle

1. **Configuration**: User creates a source configuration with params via the API
2. **Validation**: `ValidateParams` is called when saving source configuration
3. **Execution**: When a job runs, `Fetch` is called with the configured params and lookup budget
4. **Processing**: For each yielded `ImageMetadata`, `BuildUniqueID` is called to generate a stable ID
5. **Assignment**: Images are assigned to eligible devices based on matching criteria
